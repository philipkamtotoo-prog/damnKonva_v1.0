《葱の白板帝国》产品需求文档 PRD（正式版）

1. 文档信息

产品名称：葱の白板帝国
产品形态：内部使用 Web 应用
版本目标：V1
使用对象：组织内部组员
核心目标：提供一个轻量、可控的项目白板工具，支持图片生成与视频生成，并将生成结果在项目白板中统一整理、预览、备注和下载。

2. 产品背景

组织已购买视频生成 API，现需要搭建一个内部网页工具，降低组员使用门槛。该工具的定位不是复杂剪辑软件，也不是完整创作套件，而是一个轻量版生成管理白板：

组员登录后进入项目
在项目内发起图片/视频生成
生成结果自动进入白板
可在白板上对素材进行拖拽排布、文字标注、便签备注、箭头连接
可查看素材详情并下载原文件
由指定管理员账号配置 API 信息
所有功能以“简洁、可控、不乱长功能”为第一原则
3. 产品定位
3.1 产品定义

一个面向内部成员的 AI 生成白板工具，支持：

图片生成
视频生成
项目管理
素材白板整理
轻量注释
原文件下载
3.2 非目标

本产品 不承担 以下职责：

不做专业时间线剪辑
不做复杂图层系统
不做多人实时协作
不做复杂权限体系
不做批量下载
不做自动分类区域
不做自由画笔描线
不做项目搜索/排序/删除
不做项目封面与标签
4. 产品目标
4.1 业务目标

建立一个统一的内部创作入口，提升图片/视频生成流程的可管理性与素材归档效率。

4.2 用户目标

用户可以：

通过用户名密码登录
创建项目并进入白板
使用图片生成和视频生成功能
管理生成出来的素材
在白板中直观整理创意
下载原始文件
4.3 管理目标

管理员可以：

在程序内部修改登录账号密码
通过特定登录名看到 API 配置入口
在后台维护生成 API 参数
5. 用户角色

本版本不区分复杂权限角色，但存在一个隐藏的管理能力角色：

5.1 普通用户
登录系统
查看所有项目
创建项目
进入项目白板
发起图片/视频生成
查看、拖动、重命名、下载素材
添加文本框、便签、箭头连接
5.2 管理账号用户

除普通用户能力外，还可：

看到“API 配置入口”
修改 API 配置
通过程序代码维护登录名和密码
5.3 权限原则
所有人都能看到所有项目
不做项目权限隔离
API 配置入口仅某个指定登录名可见
账号密码不得在前端明文暴露，不得通过前端构建产物轻易解包读取

7. 用户流程
7.1 登录流程
用户打开系统
进入登录页
输入登录名与密码
登录成功后进入项目列表页
若登录名为指定管理员名，则在系统中出现 API 配置入口
7.2 项目使用主流程
用户在项目列表页点击“新建项目”
填写项目名称
创建成功后进入项目白板
用户在右侧面板选择“图片生成”或“视频生成”
填写 prompt 和参数后提交
系统创建生成任务
任务成功后，素材自动进入白板
用户拖动素材到任意位置
用户可添加文本框、便签、箭头关系
用户点击素材查看详情并下载原文件

9. 功能模块详细说明

功能说明

系统使用内置账号密码登录。

需求
登录表单包括：

密码
点击登录后由后端校验
登录成功后创建登录态
登录失败提示统一文案
登录态支持刷新后保留

账号密码不得写死在前端代码中

11. 交互规则
11.1 生成结果落点

新生成素材默认出现在当前白板可视区域中，避免用户找不到。

11.2 生成中卡片

提交生成后即创建占位卡片，状态显示为生成中。

11.3 右键菜单统一性

素材、文本框、便签均使用统一右键菜单逻辑：

置顶
置底
删除
11.4 便签识别

便签必须一眼可看出作者是谁，不能只显示内容。

11.5 箭头逻辑

箭头必须锚定到元素，元素移动时自动跟随，不允许脱锚。

12. 安全与技术约束
    12.1 账号密码
    必须由后端保存
    必须加密或哈希存储
    不允许出现在前端静态资源中
    不允许通过前端构建产物直接解包得到完整账号列表
    12.2 API Key
    必须只保存在后端
    必须通过服务端调用第三方接口
    前端不得直接请求第三方生成 API
    前端只调用本系统后端接口
    12.3 权限校验
    管理账号可见 API 配置入口是前端显示层校验
    API 配置读写接口必须做后端权限校验
    前后端双重限制必须同时存在
    12.4 操作日志

建议至少记录：

谁登录了
谁创建了项目
谁发起了图片/视频生成
谁修改了 API 配置
